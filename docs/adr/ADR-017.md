# ADR-017: Error Code Registry

## Status

Accepted

## Context

covguard produces structured error codes in its output that need to be:
- **Self-documenting**: Users should understand what each code means without external documentation
- **Discoverable**: CLI users need an easy way to explore codes and their remediation guidance
- **Machine-readable**: CI systems and tools need stable, parseable identifiers
- **Version-stable**: Codes should not change meaning between releases

Prior to this decision, error codes were informal strings embedded in error messages. This made it difficult for users to:
1. Look up remediation guidance for specific errors
2. Programmatically handle different error types
3. Understand the distinction between domain errors (coverage failures) and tool errors (runtime failures)

## Decision

We implement a structured error code registry with the following components:

### 1. Code Namespace Structure

Codes follow a hierarchical namespace pattern:

| Namespace | Purpose | Examples |
|-----------|---------|----------|
| `covguard.diff.*` | Domain-level findings about diff coverage | `uncovered_line`, `coverage_below_threshold`, `missing_coverage_for_file` |
| `covguard.input.*` | Input validation errors | `invalid_lcov`, `invalid_diff` |
| `tool.*` | Runtime/tool errors | `runtime_error` |

### 2. CodeInfo Registry Structure

Each code is defined in [`CODE_REGISTRY`](../../crates/covguard-types/src/lib.rs) with:

```rust
pub struct CodeInfo {
    pub code: &'static str,           // e.g., "covguard.diff.uncovered_line"
    pub name: &'static str,           // e.g., "UncoveredLine"
    pub short_description: &'static str,
    pub full_description: &'static str,
    pub remediation: &'static str,
    pub help_anchor: &'static str,    // URL anchor for docs
    pub help_uri: &'static str,       // Full documentation URL
}
```

### 3. `covguard explain` Command

The CLI provides an `explain` subcommand for code discovery:

```bash
# Explain a specific code
covguard explain covguard.diff.uncovered_line

# List all codes
covguard explain --list
```

This command:
- Looks up the code in the registry via [`explain()`](../../crates/covguard-types/src/lib.rs)
- Displays formatted output with description and remediation
- Provides a clickable link to full documentation

### 4. Documentation as Source of Truth

The [`docs/codes.md`](../codes.md) file serves as the canonical documentation for all codes:
- Each code has an HTML anchor for deep linking
- Includes meaning, common causes, and step-by-step remediation
- Registry entries link to this documentation

### 5. Code Stability Guarantee

Codes are immutable once published:
- Code strings never change meaning
- New codes may be added
- Deprecated codes remain in registry with documentation noting deprecation

## Consequences

### Positive
- **Self-documenting errors**: Users can run `covguard explain <code>` to get immediate guidance
- **CLI discoverability**: `--list` flag exposes all available codes
- **Programmable**: CI scripts can reliably parse codes for custom handling
- **Documentation sync**: Registry links directly to detailed docs
- **Onboarding**: New users learn error taxonomy through CLI exploration

### Negative
- **Maintenance overhead**: New errors require registry updates and documentation
- **Immutability constraint**: Cannot rename codes; must deprecate and add new ones
- **String bloat**: Full URLs in registry increase binary size slightly

### Neutral
- Registry lives in `covguard-types` crate for cross-crate access
- Documentation at `docs/codes.md` is the authoritative source for remediation guidance
