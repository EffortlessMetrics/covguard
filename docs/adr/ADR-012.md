# ADR-012: Deterministic Output

## Status

Accepted

## Context

covguard operates in CI/CD pipelines where reproducibility and predictability are essential:

- **Reproducible builds**: The same inputs (diff, LCOV coverage, configuration) must produce byte-identical outputs, enabling reliable caching and comparison across runs
- **CI caching**: Deterministic output allows CI systems to cache results and compare reports without false positives from ordering differences
- **Diff comparison**: When comparing reports between commits or PRs, differences should reflect actual coverage changes, not arbitrary ordering
- **Golden file testing**: Snapshot tests require stable output to avoid spurious failures and maintain test reliability
- **Debugging**: Consistent output ordering makes it easier to locate specific findings in large reports

Without deterministic output guarantees, users would experience:
- Flaky tests due to non-deterministic ordering
- Unnecessary cache invalidation in CI pipelines
- Confusion when comparing reports that should be equivalent

## Decision

covguard guarantees byte-identical output for identical inputs (diff, LCOV, configuration), excluding timing fields.

### Findings Sort Order

Findings are normatively sorted by the following keys, in priority order:

1. **Severity** (descending: error > warn > info)
2. **Path** (lexicographic, ascending)
3. **Line** (numeric ascending, None values sort last)
4. **check_id** (lexicographic)
5. **code** (lexicographic)
6. **message** (lexicographic)

This ordering is implemented in the [`sort_findings()`](../../crates/covguard-domain/src/lib.rs:290) function:

```rust
pub fn sort_findings(findings: &mut [Finding]) {
    findings.sort_by(|a, b| {
        // Severity: error > warn > info (reverse order of Ord)
        let severity_cmp = b.severity.cmp(&a.severity);
        if severity_cmp != std::cmp::Ordering::Equal {
            return severity_cmp;
        }

        // Path (lexical)
        let path_a = a.location.as_ref().map(|l| l.path.as_str()).unwrap_or("");
        let path_b = b.location.as_ref().map(|l| l.path.as_str()).unwrap_or("");
        // ... remaining sort keys
    });
}
```

### Path Normalization

All file paths in output are normalized to ensure consistent representation:

- **Repo-relative**: Paths are relative to repository root, never absolute
- **Forward slashes**: Always use `/` as separator, even on Windows
- **No `./` prefix**: Leading `./` is stripped
- **No trailing slash**: Paths never end with `/`

See [contracts/rules/paths.md](../../contracts/rules/paths.md) for full normalization rules.

### Timing Fields Excluded

The following fields vary between runs and are excluded from determinism guarantees:

- `run.started_at`
- `run.ended_at`
- `run.duration_ms`

These fields are documented in [contracts/rules/determinism.md](../../contracts/rules/determinism.md).

## Consequences

### Positive

- **Reliable golden file tests**: Snapshot tests in `crates/covguard-app/src/snapshots/` and `crates/covguard-render/src/snapshots/` produce stable, diffable output
- **CI-friendly**: Reports can be cached and compared without false differences
- **Debuggable**: Findings appear in a predictable order, making it easy to locate specific issues
- **Cross-platform**: Path normalization ensures consistent output on Windows and Unix systems
- **Schema compliance**: Deterministic ordering satisfies the schema requirements in `contracts/schemas/covguard.report.v1.json`

### Negative

- **No semantic grouping**: Findings are not grouped by file or category; users must scan the full sorted list
- **Message sensitivity**: Changes to message text (even typos) affect sort order and break golden tests
- **Locale-independent**: Lexicographic sorting uses byte comparison, not locale-aware collation

### Neutral

- The sort order prioritizes severity, so blocking issues (errors) always appear first
- BDD tests verify sorting behavior via the "findings are sorted by path then line number" step in [`crates/covguard-app/tests/bdd.rs`](../../crates/covguard-app/tests/bdd.rs:1274)
- The determinism contract is enforced through property tests and golden file comparisons
