# ADR-013: Path Normalization Strategy

## Status

Accepted

## Context

covguard matches changed lines from diffs against coverage data from LCOV reports. This matching requires consistent path representation across multiple sources:

- **Diff paths**: Git unified diff headers use `a/` and `b/` prefixes (e.g., `b/src/lib.rs`)
- **LCOV paths**: Coverage reports may contain absolute paths (e.g., `/home/user/project/src/lib.rs`), Windows paths (e.g., `C:\Users\user\project\src\lib.rs`), or paths with leading `./`
- **Cross-platform execution**: covguard runs on Windows, macOS, and Linux, each with different path separators

Without normalization, path matching fails when:

- A diff path `b/src/lib.rs` doesn't match an LCOV path `/home/user/project/src/lib.rs`
- Windows paths with backslashes don't match Unix paths with forward slashes
- Paths with `./` prefixes don't match paths without them

This would result in false negatives where changed lines appear uncovered because the coverage data couldn't be matched.

## Decision

All file paths in covguard are normalized to a canonical form before matching or output. The normalization rules are implemented in the [`covguard-paths`](../../crates/covguard-paths/src/lib.rs) crate.

### Canonical Path Form

Normalized paths have these characteristics:

1. **Repo-relative**: Paths are relative to the repository root, never absolute
2. **Forward slashes**: Always use `/` as separator, even on Windows
3. **No `./` prefix**: Leading `./` is stripped
4. **No trailing slash**: Paths never end with `/`

### Diff Path Normalization

The [`normalize_diff_path()`](../../crates/covguard-paths/src/lib.rs:17) function handles paths from unified diff headers:

```rust
pub fn normalize_diff_path(path: &str) -> String {
    let path = path.trim();
    let path = path.replace('\\', "/");
    
    let path = path
        .strip_prefix("b/")
        .or_else(|| path.strip_prefix("a/"))
        .unwrap_or(&path);
    
    path.strip_prefix("./").unwrap_or(path).to_string()
}
```

Behavior:
- Trim outer whitespace
- Convert backslashes to forward slashes
- Strip `a/` or `b/` prefix (git diff conventions)
- Strip leading `./` (once)

The [`normalize_coverage_path_with_strip()`](../../crates/covguard-paths/src/lib.rs:30) function handles LCOV `SF:` paths:

Behavior:
- Apply configured `--path-strip` prefixes first
- Strip all leading `./` sequences
- Convert backslashes to forward slashes
- Detect absolute paths and extract repo-relative portion using common source markers (`/src/`, `/lib/`, `/test/`, `/tests/`)
- Handle Windows absolute paths (e.g., `C:\...`)

### Configurable Path Stripping

Users can configure prefix stripping via `--path-strip` to handle LCOV paths that include project-specific prefixes:

```bash
covguard check --path-strip /home/user/project/ ...
```

This is documented in [contracts/rules/paths.md](../../contracts/rules/paths.md).

## Consequences

### Positive

- **Reliable diff-coverage matching**: Changed lines from diffs consistently match coverage data from LCOV reports
- **Cross-platform support**: covguard works correctly on Windows, macOS, and Linux without path-related failures
- **Deterministic output**: Normalized paths produce consistent output regardless of input path format
- **CI-friendly**: Reports are identical across different CI environments with different absolute paths
- **User-configurable**: The `--path-strip` option handles project-specific path conventions

### Negative

- **Heuristic-based absolute path handling**: The `COMMON_SOURCE_MARKERS` approach may fail for unconventional project structures that don't use `/src/`, `/lib/`, `/test/`, or `/tests/` directories
- **Potential ambiguity**: Two different files could theoretically normalize to the same path if their absolute paths differ only in a stripped prefix
- **Configuration required for edge cases**: Projects with unusual LCOV path formats may need explicit `--path-strip` configuration

### Neutral

- The normalization is lossy; original paths cannot be reconstructed from normalized paths
- Path normalization is applied at adapter boundaries (diff parsing, LCOV parsing) before data enters the domain core
- The `covguard-paths` crate is a shared dependency used by multiple adapters
